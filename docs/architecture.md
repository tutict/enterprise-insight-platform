## 摘要

FDE 交付工作台采用“项目理解 + 编译器式 AI 工程 + 可审计运行记录”的架构：

```text
项目扫描
  -> 交付简报
  -> DSL / Prompt 编译
  -> Playbook 运行
  -> Agent 生成或修改
  -> 验证与修复
  -> DeliveryRun 证据沉淀
```

核心边界：

- 前端负责配置、展示和交互。
- 后端负责项目扫描、编译、运行、事件流和持久化。
- Agent Adapter 负责模型调用、文件写入、验证命令和修复循环。
- Gateway 负责鉴权和路由保护。
